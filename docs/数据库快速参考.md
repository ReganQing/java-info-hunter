# JavaInfoHunter 数据库快速参考

## 表结构速查

### rss_sources
```sql
CREATE TABLE rss_sources (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    url VARCHAR(2048) NOT NULL UNIQUE,
    description TEXT,
    category VARCHAR(100),
    tags TEXT[],
    crawl_interval_seconds INTEGER DEFAULT 3600,
    is_active BOOLEAN DEFAULT TRUE,
    last_crawled_at TIMESTAMP,
    total_articles BIGINT DEFAULT 0,
    failed_crawls BIGINT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### raw_content
```sql
CREATE TABLE raw_content (
    id BIGSERIAL PRIMARY KEY,
    rss_source_id BIGINT REFERENCES rss_sources(id) ON DELETE CASCADE,
    guid VARCHAR(255) NOT NULL,
    title VARCHAR(500) NOT NULL,
    link VARCHAR(2048),
    raw_content TEXT NOT NULL,
    content_hash VARCHAR(64) UNIQUE,
    publish_date TIMESTAMP,
    crawl_date TIMESTAMP DEFAULT NOW(),
    processing_status VARCHAR(20) DEFAULT 'pending',
    embedding vector(1536),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    UNIQUE (guid, rss_source_id)
);
```

### news
```sql
CREATE TABLE news (
    id BIGSERIAL PRIMARY KEY,
    raw_content_id BIGINT UNIQUE REFERENCES raw_content(id) ON DELETE CASCADE,
    title VARCHAR(500) NOT NULL,
    summary TEXT NOT NULL,
    full_content TEXT,
    topics TEXT[],
    keywords TEXT[],
    sentiment VARCHAR(20),
    sentiment_score DECIMAL(3,2),
    importance_score DECIMAL(3,2),
    category VARCHAR(100),
    tags TEXT[],
    language VARCHAR(10) DEFAULT 'en',
    slug VARCHAR(255) UNIQUE,
    view_count BIGINT DEFAULT 0,
    like_count INTEGER DEFAULT 0,
    is_published BOOLEAN DEFAULT FALSE,
    is_featured BOOLEAN DEFAULT FALSE,
    published_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### agent_executions
```sql
CREATE TABLE agent_executions (
    id BIGSERIAL PRIMARY KEY,
    agent_id VARCHAR(100) NOT NULL,
    agent_type VARCHAR(50) NOT NULL,
    execution_id VARCHAR(100) UNIQUE,
    task_type VARCHAR(100),
    coordination_pattern VARCHAR(50),
    input_data JSONB,
    output_data JSONB,
    status VARCHAR(20) NOT NULL,
    start_time TIMESTAMP DEFAULT NOW(),
    end_time TIMESTAMP,
    duration_milliseconds INTEGER,
    tokens_used INTEGER,
    estimated_cost_usd DECIMAL(10,4),
    raw_content_id BIGINT REFERENCES raw_content(id) ON DELETE SET NULL,
    news_id BIGINT REFERENCES news(id) ON DELETE SET NULL,
    correlation_id VARCHAR(100),
    parent_execution_id VARCHAR(100),
    retry_count INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

## 常用查询

### 查找待爬取的源
```sql
SELECT * FROM rss_sources
WHERE is_active = TRUE
  AND (last_crawled_at IS NULL OR last_crawled_at < NOW() - INTERVAL '1 hour')
ORDER BY last_crawled_at ASC NULLS LAST;
```

### 查找待处理内容
```sql
SELECT rc.*, rs.name AS source_name
FROM raw_content rc
JOIN rss_sources rs ON rc.rss_source_id = rs.id
WHERE rc.processing_status = 'pending'
ORDER BY rc.crawl_date ASC
LIMIT 100;
```

### 向量相似度搜索
```sql
SELECT id, title,
       1 - (embedding <=> '[0.1, 0.2, ...]') AS similarity
FROM raw_content
WHERE embedding IS NOT NULL
ORDER BY embedding <=> '[0.1, 0.2, ...]'
LIMIT 10;
```

### 全文搜索
```sql
SELECT id, title, summary
FROM news
WHERE to_tsvector('english', title || ' ' || summary)
      @@ to_tsquery('english', 'machine & learning')
ORDER BY ts_rank(to_tsvector('english', title || ' ' || summary),
                 to_tsquery('english', 'machine & learning')) DESC
LIMIT 20;
```

### Agent 执行统计
```sql
SELECT
    agent_id,
    COUNT(*) AS total,
    COUNT(*) FILTER (WHERE status = 'completed') AS completed,
    COUNT(*) FILTER (WHERE status = 'failed') AS failed,
    AVG(duration_milliseconds) AS avg_duration
FROM agent_executions
WHERE start_time > NOW() - INTERVAL '7 days'
GROUP BY agent_id;
```

## 索引列表

| 表名 | 索引名 | 类型 | 字段 |
|------|--------|------|------|
| rss_sources | idx_rss_sources_active | B-Tree | is_active |
| rss_sources | idx_rss_sources_tags | GIN | tags |
| raw_content | idx_raw_content_embedding | IVFFlat | embedding |
| raw_content | idx_raw_content_status | B-Tree | processing_status |
| news | idx_news_published | B-Tree | is_published, published_at |
| news | idx_news_fulltext | GIN | to_tsvector(...) |
| news | idx_news_tags | GIN | tags |
| agent_executions | idx_agent_executions_agent_time | B-Tree | agent_id, start_time |

## 约束列表

| 表名 | 约束类型 | 字段 | 说明 |
|------|----------|------|------|
| rss_sources | UNIQUE | url | 防止重复源 |
| raw_content | UNIQUE | content_hash | 全局去重 |
| raw_content | UNIQUE | guid + rss_source_id | 源内去重 |
| news | UNIQUE | slug | URL slug 唯一 |
| agent_executions | UNIQUE | execution_id | 执行 ID 唯一 |

## 枚举值

### processing_status
- `pending`: 待处理
- `processing`: 处理中
- `completed`: 已完成
- `failed`: 失败

### sentiment
- `positive`: 积极
- `neutral`: 中性
- `negative`: 消极

### execution_status
- `running`: 运行中
- `completed`: 已完成
- `failed`: 失败
- `cancelled`: 已取消

### coordination_pattern
- `chain`: 链式
- `parallel`: 并行
- `master_worker`: 主从
- `standalone`: 独立

## 快速命令

### 连接数据库
```bash
psql -U postgres -d javainfohunter
```

### 查看表结构
```sql
\d table_name
```

### 查看索引
```sql
\di table_name*
```

### 查看表大小
```sql
SELECT
    tablename,
    pg_size_pretty(pg_total_relation_size(tablename::regclass)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(tablename::regclass) DESC;
```

### 查看慢查询
```sql
SELECT query, mean_exec_time, calls
FROM pg_stat_statements
ORDER BY mean_exec_time DESC
LIMIT 10;
```

### 备份数据库
```bash
pg_dump -U postgres -d javainfohunter -F c -f backup.dump
```

### 恢复数据库
```bash
pg_restore -U postgres -d javainfohunter --clean --if-exists backup.dump
```
