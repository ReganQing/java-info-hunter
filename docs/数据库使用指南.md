# JavaInfoHunter 数据库使用指南

## 快速开始

### 1. 安装 PostgreSQL 和 pgvector

**macOS (Homebrew)**:
```bash
brew install postgresql@16
brew install pgvector
```

**Ubuntu/Debian**:
```bash
sudo apt-get install postgresql-16 postgresql-contrib-16
# 安装 pgvector (需要从源码编译)
git clone --branch v0.5.1 https://github.com/pgvector/pgvector.git
cd pgvector
make
sudo make install
```

**Windows**:
- 下载 PostgreSQL 安装程序: https://www.postgresql.org/download/windows/
- pgvector 需要从源码编译或使用 WSL2

### 2. 创建数据库

```bash
# 连接到 PostgreSQL
psql -U postgres

# 创建数据库
CREATE DATABASE javainfohunter;

# 连接到数据库
\c javainfohunter

# 启用 pgvector 扩展
CREATE EXTENSION IF NOT EXISTS vector;

# 验证扩展
\dx vector
```

### 3. 配置应用连接

编辑 `src/main/resources/application.properties`:

```properties
spring.datasource.url=jdbc:postgresql://localhost:5432/javainfohunter
spring.datasource.username=postgres
spring.datasource.password=your_password
```

或使用环境变量：

```bash
export DB_PASSWORD=your_password
```

### 4. 运行数据库迁移

应用启动时，Flyway 会自动执行迁移脚本：

```bash
mvnw.cmd spring-boot:run
```

或手动执行迁移：

```bash
mvnw.cmd flyway:migrate
```

### 5. 验证安装

```bash
# 检查表是否创建成功
psql -U postgres -d javainfohunter -c "\dt"

# 检查索引
psql -U postgres -d javainfohunter -c "\di"

# 查看示例数据
psql -U postgres -d javainfohunter -c "SELECT * FROM rss_sources;"
```

## Repository 使用示例

### RssSourceRepository

```java
@Autowired
private RssSourceRepository rssSourceRepository;

// 查找所有活跃的 RSS 源
List<RssSource> activeSources = rssSourceRepository.findByIsActiveTrue();

// 查找待爬取的源
Instant threshold = Instant.now().minusSeconds(3600);
List<RssSource> dueSources = rssSourceRepository.findSourcesDueForCrawling(threshold);

// 按分类查找
List<RssSource> techSources = rssSourceRepository.findByCategory("technology");

// 添加新 RSS 源
RssSource newSource = RssSource.builder()
    .name("My Tech Blog")
    .url("https://example.com/feed")
    .category("technology")
    .tags(Arrays.asList("tech", "programming"))
    .crawlIntervalSeconds(3600)
    .isActive(true)
    .build();
rssSourceRepository.save(newSource);
```

## 性能优化建议

### 1. 批量操作

```java
// 批量插入
List<RawContent> contents = // ... 获取列表
rawContentRepository.saveAll(contents);

// 批量更新状态
List<Long> ids = contents.stream().map(RawContent::getId).collect(Collectors.toList());
rawContentRepository.bulkUpdateProcessingStatus(
    ids,
    RawContent.ProcessingStatus.PROCESSING
);
```

### 2. 分页查询

```java
// 使用游标分页（避免 OFFSET）
Page<News> page = newsRepository.findByIsPublishedTrueOrderByPublishedAtDesc(
    PageRequest.of(0, 20)
);

// 获取下一页
if (page.hasNext()) {
    Page<News> nextPage = newsRepository.findByIsPublishedTrueOrderByPublishedAtDesc(
        page.nextPageable()
    );
}
```

### 3. 延迟加载优化

```java
// 使用 EntityGraph 避免 N+1 查询
@EntityGraph(attributePaths = {"rssSource"})
List<RawContent> findByProcessingStatus(ProcessingStatus status);
```

### 4. 索引优化

```sql
-- 创建覆盖索引
CREATE INDEX idx_raw_content_covering
ON raw_content(rss_source_id, processing_status, id)
INCLUDE (title, crawl_date);

-- 创建部分索引（只索引活跃数据）
CREATE INDEX idx_news_published_recent
ON news(published_at DESC)
WHERE is_published = TRUE
  AND published_at > CURRENT_DATE - INTERVAL '30 days';
```

## 监控和调试

### 1. 启用 SQL 日志

```properties
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
logging.level.org.hibernate.SQL=DEBUG
logging.level.org.hibernate.type.descriptor.sql.BasicBinder=TRACE
```

### 2. 查看查询计划

```sql
EXPLAIN ANALYZE
SELECT * FROM news
WHERE is_published = TRUE
ORDER BY published_at DESC
LIMIT 20;
```

### 3. 监控慢查询

```sql
-- 查看慢查询配置
SHOW log_min_duration_statement;

-- 设置慢查询阈值（1 秒）
ALTER SYSTEM SET log_min_duration_statement = 1000;

-- 重新加载配置
SELECT pg_reload_conf();
```

### 4. 检查索引使用情况

```sql
-- 查看未使用的索引
SELECT schemaname, tablename, indexname, idx_scan
FROM pg_stat_user_indexes
WHERE idx_scan = 0
  AND indexname NOT LIKE '%_pkey'
ORDER BY pg_relation_size(indexrelid) DESC;
```

## 常见问题

### Q1: Flyway 迁移失败

**问题**: 迁移脚本执行失败

**解决方案**:
```bash
# 修复迁移脚本后，手动修复 Flyway 状态
mvnw.cmd flyway:repair

# 或重新创建数据库
psql -U postgres -c "DROP DATABASE javainfohunter;"
psql -U postgres -c "CREATE DATABASE javainfohunter;"
```

### Q2: pgvector 扩展不存在

**问题**: `ERROR: extension "vector" does not exist`

**解决方案**:
```bash
# 安装 pgvector（见上文安装说明）
# 然后创建扩展
psql -U postgres -d javainfohunter -c "CREATE EXTENSION IF NOT EXISTS vector;"
```

### Q3: 连接池耗尽

**问题**: `Connection pool exhausted`

**解决方案**:
```properties
# 增加连接池大小
spring.datasource.hikari.maximum-pool-size=50
spring.datasource.hikari.minimum-idle=10

# 或启用连接泄漏检测
spring.datasource.hikari.leak-detection-threshold=60000
```

### Q4: 向量索引不生效

**问题**: 向量查询没有使用索引

**解决方案**:
```sql
-- 确保有足够的数据（IVFFlat 需要至少 1000 行）
-- 检查索引是否被使用
EXPLAIN ANALYZE
SELECT * FROM raw_content
ORDER BY embedding <=> '[0.1, 0.2, ...]'
LIMIT 10;

-- 如果没有使用索引，强制使用
SET enable_seqscan = off;
```

## 最佳实践

### 1. 事务管理

```java
@Transactional
public void processContent(Long contentId) {
    RawContent content = rawContentRepository.findById(contentId).orElseThrow();
    content.markAsProcessing();
    rawContentRepository.save(content);

    // 处理内容...

    content.markAsCompleted();
    rawContentRepository.save(content);
}
```

### 2. 异常处理

```java
try {
    RawContent content = rawContentRepository.findById(id).orElseThrow();
    // 处理内容...
} catch (EntityNotFoundException e) {
    log.error("Content not found: {}", id);
    throw new ResourceNotFoundException("Content not found");
} catch (DataAccessException e) {
    log.error("Database error", e);
    throw new ServiceException("Database operation failed");
}
```

### 3. 缓存策略

```java
@Cacheable(value = "rssSources", key = "#id")
public RssSource findById(Long id) {
    return rssSourceRepository.findById(id).orElse(null);
}

@CacheEvict(value = "rssSources", key = "#source.id")
public void updateSource(RssSource source) {
    rssSourceRepository.save(source);
}
```

```java
// 启用 JPA Auditing

}
```
