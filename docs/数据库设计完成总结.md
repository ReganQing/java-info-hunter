## 关键设计亮点

### 1. 数据库三范式
- ✅ 第一范式：所有字段都是原子的
- ✅ 第二范式：非主键字段完全依赖于主键
- ✅ 第三范式：非主键字段不传递依赖于主键

### 2. 索引优化
- **B-Tree 索引**: 用于等值查询、范围查询、排序
- **GIN 索引**: 用于数组查询、全文搜索
- **IVFFlat 索引**: 用于向量相似度搜索（近似最近邻）
- **部分索引**: 只索引满足条件的行，减少维护开销
- **复合索引**: 优化多列查询

### 3. 向量搜索
- 使用 pgvector 扩展
- IVFFlat 索引（lists = 100）
- 余弦距离（vector_cosine_ops）
- 支持 1536 维向量（OpenAI embedding）

### 4. 全文搜索
- 使用 PostgreSQL 内置的全文搜索
- GIN 索引优化搜索性能
- 支持英文文本搜索（可扩展到其他语言）

### 5. 数据完整性
- 主键约束：保证记录唯一性
- 外键约束：保证引用完整性
- 唯一约束：防止业务数据重复
- 检查约束：验证数据有效性
- 触发器：自动维护时间戳

### 6. 扩展性设计
- 预留 tags[] 数组字段，支持灵活扩展
- JSONB 字段支持动态 schema
- 预留分区策略（按时间分区）
- 预留读写分离（主从架构）

### 7. 性能优化
- HikariCP 连接池（高性能）
- 批量操作支持（JPA batch_size）
- 延迟加载（避免 N+1 查询）
- 查询缓存（可选）
- 读写分离（未来）
