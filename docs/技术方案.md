# JavaInfoHunter 全网资讯猎手技术方案

## 1. 项目概述

### 1.1 项目简介
JavaInfoHunter 是一个基于 Java 21 和 Spring Boot 4.0 构建的高性能分布式资讯采集与智能分析系统。系统通过虚拟线程实现高并发抓取，结合消息队列解耦处理流程，利用 **Spring AI + Agent 编排模式**进行内容分析与向量化存储，最终实现全网资讯的实时监测与智能推荐。

### 1.2 核心价值
- **高性能**: 基于 JDK 21 虚拟线程，单机可支撑数万级并发请求
- **高可靠**: 消息队列 + 手动 ACK 机制保证数据零丢失
- **智能化**: 集成 **Spring AI + Agent 编排模式**，实现多智能体协作的内容分析与处理
- **可扩展**: 微服务架构，各组件可独立水平扩展
- **Agent 编排**: 采用 ReAct 模式的多智能体协作，支持 Master-Worker、Chain、Parallel 等协调模式

## 3. Agent 编排架构设计

### 3.1 Agent 编排模式

JavaInfoHunter 采用 **Agent 编排模式** 作为核心智能处理框架，使用 javainfohunter-ai-service 模块。

#### 3.1.1 Agent 层次结构

```
┌─────────────────────────────────────────────────────────────┐
│                    BaseAgent (基础抽象)                      │
│  • 状态管理 (IDLE/RUNNING/FINISHED/ERROR)                    │
│  • 生命周期管理                                              │
│  • 消息记忆                                                  │
│  • 步骤控制                                                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                  ReActAgent (思考-行动循环)                  │
│  • think(): 思考是否需要行动                                 │
│  • act(): 执行具体行动                                       │
│  • step(): 完整的 think-act 循环                             │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              ToolCallAgent (工具调用增强)                     │
│  • Spring AI ChatClient 集成                                 │
│  • 工具回调管理                                              │
│  • 工具执行结果处理                                          │
│  • 自动终止检测                                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│            Specialized Agents (专用 Agent)                   │
│  • CrawlerAgent      (内容抓取与分析)                         │
│  • AnalysisAgent     (深度内容分析)                           │
│  • SummaryAgent      (智能摘要生成)                           │
│  • ClassificationAgent (内容分类)                             │
│  • RecommendationAgent (智能推荐)                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3.1.2 协调模式

支持四种多智能体协作模式：

**1. Master-Worker 模式** ⏳ 计划中
```
Master Agent (CoordinatorAgent)
    ↓ 分配任务
    ├─→ Worker Agent 1 (CrawlerAgent)
    ├─→ Worker Agent 2 (AnalysisAgent)
    ├─→ Worker Agent 3 (SummaryAgent)
    └─→ Worker Agent 4 (ClassificationAgent)
    ↓ 汇总结果
Master Agent (生成最终报告)
```

**2. Chain 模式** ✅ 已实现
```
Task → CrawlerAgent → AnalysisAgent → SummaryAgent → RecommendationAgent → Result
       (抓取)          (分析)          (摘要)          (推荐)              (最终结果)
```

**3. Parallel 模式** ✅ 已实现
```
                              Task
                    ┌──────────┼──────────┐
                    ↓          ↓          ↓
              Agent 1     Agent 2     Agent 3
                    └──────────┼──────────┘
                        ↓ Aggregated
                         Result
```

**4. 自定义模式**
支持通过实现 `CollaborationPattern` 接口扩展自定义协调模式。

### 3.2 JavaInfoHunter Agent 架构

#### 3.2.1 Agent 角色定义

| Agent 名称 | 父类 | 功能描述 | 工具集 |

| **CrawlerAgent** | ToolCallAgent | 从消息队列消费内容，解析 HTML/RSS，提取结构化数据 | HtmlParserTool, RssParserTool, ContentExtractorTool |

## 5. 技术架构

### 5.1 核心技术要点

#### 5.1.1 虚拟线程并发模型
```java
// 传统线程池 vs 虚拟线程对比
// 传统: 200 个平台线程，内存占用 ~1GB
ExecutorService传统 = Executors.newFixedThreadPool(200);

// 虚拟线程: 10 万个虚拟线程，内存占用 ~100MB
ExecutorService虚拟 = Executors.newVirtualThreadPerTaskExecutor();
```

**优势:**
- 轻量级: 每个虚拟线程仅占用几 KB 内存
- 高并发: 单机可创建百万级线程
- 阻塞友好: 传统阻塞代码无需改造

#### 5.1.2 Spring AI 集成
```java
// Spring AI ChatClient 配置
@Bean
public ChatClient chatClient(ChatModel chatModel) {
    return ChatClient.builder(chatModel)
        .defaultAdvisors(
            new MessageChatMemoryAdvisor(chatMemory),
            new QuestionAnswerAdvisor(vectorStore)
        )
        .build();
}

// Agent 中使用 Spring AI
@Override
public boolean think() {
    ChatResponse response = chatClient.prompt()
        .system(getSystemPrompt())
        .user(getNextStepPrompt())
        .toolCallbacks(availableTools)
        .call()
        .chatResponse();

    // 处理响应和工具调用
    return response.hasToolCalls();
}
```

#### 5.1.4 消息队列可靠性保障
```java
@RabbitListener(queues = "crawler.queue")
public void handle(String message, Channel channel,
                   @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
    try {
        // 交给 CrawlerAgent 处理
        crawlerAgent.run(message);
        // 手动 ACK
        channel.basicAck(deliveryTag, false);
    } catch (Exception e) {
        // 拒绝并重新入队
        channel.basicNack(deliveryTag, false, true);
    }
}
```

**关键机制:**
- Manual ACK: 确保消息处理成功后才确认
- 持久化: Queue + Message 双重持久化
- 重试机制: 失败消息自动重新入队

#### 5.1.5 向量存储与检索
```sql
-- pgvector 向量相似度搜索
SELECT title, summary,
       embedding <=> '[0.1, 0.2, ...]' AS distance
FROM news
ORDER BY distance
LIMIT 10;
```

**应用场景:**
- 相似新闻推荐
- 内容去重
- 语义搜索

### 6.2 功能详细说明

#### 6.2.1 核心功能模块

| 模块 | 功能 | Agent 协作 | 技术实现 |
|------|------|-----------|----------|
| **URL 管理** | RSS 源配置、爬取调度、频率控制 | 单 Agent | Spring Scheduler + Redis |
| **内容采集** | 多源并发抓取、HTML 解析、正文提取 | CrawlerAgent | Virtual Threads + Jsoup |
| **内容处理** | 去重清洗、AI 分析、内容分类 | Multi-Agent (Parallel) | Agent 协作 + Spring AI |
| **向量存储** | 文本向量化、相似度计算、语义索引 | CoordinatorAgent | PostgreSQL + pgvector |

#### 6.2.2 Agent 协作场景

**场景 1: 新闻内容处理**
```
Pattern: Parallel (并行处理)
Input: 原始新闻内容
Agents:
  - AnalysisAgent: 情感分析、关键词提取
  - SummaryAgent: 生成摘要、提取观点
  - ClassificationAgent: 内容分类、标签生成
Output: 结构化分析结果
```

**场景 2: 个性化推荐**
```
Pattern: Chain (链式处理)
Input: 用户 ID
Chain:
  1. UserProfileAgent: 分析用户偏好
  2. VectorSearchAgent: 向量检索相似内容
  3. RankingAgent: 排序并生成推荐列表
Output: 个性化推荐结果
```

**场景 3: 热点追踪**
```
Pattern: Master-Worker (主从模式)
Master: TrendingCoordinatorAgent
Workers:
  - CrawlerAgent: 实时抓取热点内容
  - AnalysisAgent: 分析热度趋势
  - AlertAgent: 触发热点告警
Output: 热点趋势报告
```

## 8. 数据模型设计

**注意：** 以下数据库设计为 Phase 2 计划内容，当前版本未实现数据库层。当前实现专注于 AI 编排能力，数据持久化将在后续版本中添加。

### 8.1 核心数据表

```sql
-- RSS 订阅源表
CREATE TABLE rss_sources (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    url VARCHAR(1024) NOT NULL UNIQUE,
    category VARCHAR(100),
    crawl_interval INT DEFAULT 3600,  -- 爬取间隔(秒)
    active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 原始内容表
CREATE TABLE raw_content (
    id BIGSERIAL PRIMARY KEY,
    source_id BIGINT REFERENCES rss_sources(id),
    url VARCHAR(1024) NOT NULL UNIQUE,
    title VARCHAR(512),
    content TEXT,
    author VARCHAR(255),
    publish_time TIMESTAMP,
    crawled_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(50) DEFAULT 'pending'  -- pending, processing, completed, failed
);

-- 处理后的新闻表
CREATE TABLE news (
    id BIGSERIAL PRIMARY KEY,
    raw_content_id BIGINT REFERENCES raw_content(id),
    title VARCHAR(512) NOT NULL,
    summary TEXT,                    -- AI 生成的摘要
    summary_en TEXT,                 -- 英文摘要
    key_points TEXT[],               -- 关键观点数组
    content TEXT,                    -- 清洗后的正文
    category VARCHAR(100),           -- 分类
    tags VARCHAR(512)[],             -- 标签数组
    sentiment VARCHAR(20),           -- 情感倾向
    keywords VARCHAR(512)[],         -- 关键词数组
    topics VARCHAR(512)[],           -- 主题数组
    embedding vector(1536),          -- 向量表示
    publish_time TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Agent 执行记录表
CREATE TABLE agent_executions (
    id BIGSERIAL PRIMARY KEY,
    agent_name VARCHAR(255) NOT NULL,
    agent_type VARCHAR(100),         -- Crawler, Analysis, Summary, etc.
    coordination_pattern VARCHAR(50), -- Master-Worker, Chain, Parallel
    task_description TEXT,
    input_data JSONB,
    output_data JSONB,
    status VARCHAR(50),              -- RUNNING, FINISHED, ERROR
    steps INT,
    duration_ms BIGINT,
    error_message TEXT,
    started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP
);

-- 创建向量相似度索引
CREATE INDEX idx_news_embedding ON news USING ivfflat (embedding vector_cosine_ops);
```
