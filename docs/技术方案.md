# JavaInfoHunter 全网资讯猎手技术方案

## 1. 项目概述

### 1.1 项目简介
JavaInfoHunter 是一个基于 Java 21 和 Spring Boot 4.0 构建的高性能分布式资讯采集与智能分析系统。系统通过虚拟线程实现高并发抓取，结合消息队列解耦处理流程，利用 **Spring AI + Agent 编排模式**进行内容分析与向量化存储，最终实现全网资讯的实时监测与智能推荐。

### 1.2 核心价值
- **高性能**: 基于 JDK 21 虚拟线程，单机可支撑数万级并发请求
- **高可靠**: 消息队列 + 手动 ACK 机制保证数据零丢失
- **智能化**: 集成 **Spring AI + Agent 编排模式**，实现多智能体协作的内容分析与处理
- **可扩展**: 微服务架构，各组件可独立水平扩展
- **Agent 编排**: 采用 ReAct 模式的多智能体协作，支持 Master-Worker、Chain、Parallel 等协调模式

## 3. Agent 编排架构设计

### 3.1 Agent 编排模式

JavaInfoHunter 采用 **Agent 编排模式** 作为核心智能处理框架，使用 javainfohunter-ai-service 模块。

#### 3.1.1 Agent 层次结构

```
┌─────────────────────────────────────────────────────────────┐
│                    BaseAgent (基础抽象)                      │
│  • 状态管理 (IDLE/RUNNING/FINISHED/ERROR)                    │
│  • 生命周期管理                                              │
│  • 消息记忆                                                  │
│  • 步骤控制                                                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                  ReActAgent (思考-行动循环)                  │
│  • think(): 思考是否需要行动                                 │
│  • act(): 执行具体行动                                       │
│  • step(): 完整的 think-act 循环                             │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              ToolCallAgent (工具调用增强)                     │
│  • Spring AI ChatClient 集成                                 │
│  • 工具回调管理                                              │
│  • 工具执行结果处理                                          │
│  • 自动终止检测                                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│            Specialized Agents (专用 Agent)                   │
│  • CrawlerAgent      (内容抓取与分析)                         │
│  • AnalysisAgent     (深度内容分析)                           │
│  • SummaryAgent      (智能摘要生成)                           │
│  • ClassificationAgent (内容分类)                             │
│  • RecommendationAgent (智能推荐)                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3.1.2 协调模式

支持四种多智能体协作模式：

**1. Master-Worker 模式** ⏳ 计划中
```
Master Agent (CoordinatorAgent)
    ↓ 分配任务
    ├─→ Worker Agent 1 (CrawlerAgent)
    ├─→ Worker Agent 2 (AnalysisAgent)
    ├─→ Worker Agent 3 (SummaryAgent)
    └─→ Worker Agent 4 (ClassificationAgent)
    ↓ 汇总结果
Master Agent (生成最终报告)
```

**2. Chain 模式** ✅ 已实现
```
Task → CrawlerAgent → AnalysisAgent → SummaryAgent → RecommendationAgent → Result
       (抓取)          (分析)          (摘要)          (推荐)              (最终结果)
```

**3. Parallel 模式** ✅ 已实现
```
                              Task
                    ┌──────────┼──────────┐
                    ↓          ↓          ↓
              Agent 1     Agent 2     Agent 3
                    └──────────┼──────────┘
                        ↓ Aggregated
                         Result
```

**4. 自定义模式**
支持通过实现 `CollaborationPattern` 接口扩展自定义协调模式。

### 3.2 JavaInfoHunter Agent 架构

## 5. 技术架构

### 5.1 核心技术要点

#### 5.1.1 虚拟线程并发模型
```java
// 传统线程池 vs 虚拟线程对比
// 传统: 200 个平台线程，内存占用 ~1GB
ExecutorService传统 = Executors.newFixedThreadPool(200);

// 虚拟线程: 10 万个虚拟线程，内存占用 ~100MB
ExecutorService虚拟 = Executors.newVirtualThreadPerTaskExecutor();
```

**优势:**
- 轻量级: 每个虚拟线程仅占用几 KB 内存
- 高并发: 单机可创建百万级线程
- 阻塞友好: 传统阻塞代码无需改造

#### 5.1.2 Spring AI 集成
```java
// Spring AI ChatClient 配置
@Bean
public ChatClient chatClient(ChatModel chatModel) {
    return ChatClient.builder(chatModel)
        .defaultAdvisors(
            new MessageChatMemoryAdvisor(chatMemory),
            new QuestionAnswerAdvisor(vectorStore)
        )
        .build();

// Agent 中使用 Spring AI
@Override
public boolean think() {
    ChatResponse response = chatClient.prompt()
        .system(getSystemPrompt())
        .user(getNextStepPrompt())
        .toolCallbacks(availableTools)
        .call()
        .chatResponse();

    // 处理响应和工具调用
    return response.hasToolCalls();
}
```

#### 5.1.4 消息队列可靠性保障
```java
@RabbitListener(queues = "crawler.queue")
public void handle(String message, Channel channel,
                   @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
    try {
        // 交给 CrawlerAgent 处理
        crawlerAgent.run(message);
        // 手动 ACK
        channel.basicAck(deliveryTag, false);
    } catch (Exception e) {
        // 拒绝并重新入队
        channel.basicNack(deliveryTag, false, true);
    }
}
```

**关键机制:**
- Manual ACK: 确保消息处理成功后才确认
- 持久化: Queue + Message 双重持久化
- 重试机制: 失败消息自动重新入队

#### 5.1.5 向量存储与检索
```sql
-- pgvector 向量相似度搜索
SELECT title, summary,
       embedding <=> '[0.1, 0.2, ...]' AS distance
FROM news
ORDER BY distance
LIMIT 10;
```

**应用场景:**
- 相似新闻推荐
- 内容去重
- 语义搜索

#### 6.2.2 Agent 协作场景

**场景 1: 新闻内容处理**
```
Pattern: Parallel (并行处理)

**场景 3: 热点追踪**
```
Pattern: Master-Worker (主从模式)
Master: TrendingCoordinatorAgent
Workers:
  - CrawlerAgent: 实时抓取热点内容

## 8. 数据模型设计

### 8.1 数据库架构概述

JavaInfoHunter 使用 **PostgreSQL 16 + pgvector** 作为主数据库，配合 Flyway 进行版本管理。数据库设计遵循第三范式，优化查询性能，并支持 AI 原生特性（向量搜索、JSONB 存储）。

**技术栈：**
- **数据库**: PostgreSQL 16+
- **向量扩展**: pgvector（用于语义搜索）
- **迁移工具**: Flyway
- **ORM**: Spring Data JPA (Hibernate)
- **连接池**: HikariCP

### 8.2 核心数据表

#### 8.2.1 RSS 订阅源表 (rss_sources)

**用途**: 存储 RSS 订阅源信息及爬取配置

**核心字段：**
- `id` (BIGSERIAL, PK): 主键
- `name` (VARCHAR 255): 源名称
- `url` (VARCHAR 2048, UNIQUE): RSS feed URL，唯一约束
- `category` (VARCHAR 100): 分类（如 technology, business）
- `tags[]` (TEXT[]): 标签数组
- `is_active` (BOOLEAN): 是否活跃爬取

- `total_articles` (BIGINT): 总文章数

**索引：**

### 9.1 架构概述

JavaInfoHunter 采用 **消息队列驱动的 Agent 编排架构**，RabbitMQ 作为核心消息中间件，承担以下关键职责：

```

│     - 爬虫模块持续推送 URL 和原始内容                         │
│     - 处理模块按需消费，避免过载                               │
│                                                              │
│  2. 异步通信与事件驱动                                        │
│     - Agent 之间通过消息异步通信                              │
│     - 事件驱动的处理流程                                      │
│                                                              │
│  3. 负载均衡与并发控制                                        │
│     - 多个消费者实例竞争消费                                  │
│     - 自动分配任务，实现水平扩展                              │
│                                                              │
│  4. 可靠性保障                                                │
│     - 消息持久化，防止数据丢失                                │
│     - 死信队列处理失败消息                                    │
│     - 重试机制保证最终一致性                                  │
└─────────────────────────────────────────────────────────────┘
```

### 9.2 Exchange 和 Queue 设计

#### 9.2.1 Exchange（交换机）设计

| Exchange 名称 | 类型 | 用途 | 绑定策略 |
|--------------|------|------|----------|
| `crawler.direct` | Direct | 爬虫任务分发 | 按 routing key 精确路由到不同爬虫队列 |
| `content.fanout` | Fanout | 内容广播 | 广播到所有内容处理队列 |
| `agent.topic` | Topic | Agent 任务路由 | 按模式匹配路由到不同 Agent 队列 |
| `dlx.exchange` | Direct | 死信处理 | 路由失败消息到死信队列 |

#### 9.2.2 Queue（队列）设计

| Queue 名称 | 绑定 Exchange | Routing Key | 用途 |
|-----------|--------------|-------------|------|
| `crawler.url.queue` | crawler.direct | `crawler.url` | 待爬取 URL 队列 |
| `crawler.content.queue` | crawler.direct | `crawler.content` | 原始内容队列 |
| `process.pending.queue` | content.fanout | - | 待处理内容队列 |
| `agent.crawler.queue` | agent.topic | `agent.crawler.*` | CrawlerAgent 任务队列 |
| `agent.analysis.queue` | agent.topic | `agent.analysis.*` | AnalysisAgent 任务队列 |
| `agent.summary.queue` | agent.topic | `agent.summary.*` | SummaryAgent 任务队列 |
| `agent.classification.queue` | agent.topic | `agent.classification.*` | ClassificationAgent 任务队列 |
| `dlq.queue` | dlx.exchange | `#` | 死信队列（处理失败消息） |

                           ↓

@RabbitListener(queues = "crawler.url.queue")
public void handleUrlTask(
    CrawlerUrlMessage message,

### 9.6 与 Agent 编排的集成

#### 9.6.1 消息驱动的 Agent 触发

```java
@Service
public class MessageDrivenAgentService {

    @Autowired
    private TaskCoordinator taskCoordinator;

    @RabbitListener(queues = "process.pending.queue")
    public void handleContentProcessing(ContentMessage message) {
        // 使用 Agent 协编排处理内容
        CoordinationResult result = taskCoordinator.executeParallel(
            "处理内容: " + message.getTitle(),
            List.of(
                "analysis-agent",
                "summary-agent",
                "classification-agent"
            )
        );

        // 发送处理结果到下一个队列
        if (result.isSuccess()) {
            sendToProcessedQueue(result);
        } else {
            sendToDeadLetterQueue(message, result.getError());
        }
    }
}
```

#### 9.6.2 Agent 执行结果到消息队列

```java
public class MessageProducingAgent extends ToolCallAgent {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    @Override
    protected void onComplete(ExecutionContext context) {
        super.onComplete(context);

        // 发送完成消息
        AgentCompletionMessage message = AgentCompletionMessage.builder()
            .agentId(getAgentId())
            .executionId(context.getExecutionId())
            .result(context.getOutput())
            .build();

        rabbitTemplate.convertAndSend("agent.results", message);
    }
}
```

### 9.7 监控和告警

#### 9.7.1 队列监控指标

- **队列深度**: 队列中消息数量
- **消费速率**: 消息/秒
- **错误率**: 失败消息占比
- **延迟**: 消息从生产到消费的时间
